# 核心精要

本节汇集了技术文档写作中最核心的原则。在编写任何技术内容时，遵循这些准则不仅能确保信息传递的精准与专业，更能帮助你构建起与读者之间清晰、高效的沟通桥梁。

## 1. 语气和内容

### 1.1 保持对话感与亲和力

语言要像朋友交谈一样自然，但切记不可轻浮，避免过度开玩笑或使用非正式的俚语。

### 1.2 杜绝“预告式”写作

不要在文档中预告未来的功能（如“即将推出...”），只写目前已有的事实。

### 1.3 使用描述性链接文本

严禁使用“点击这里”或“阅读更多”。链接文字应直接描述目标内容，例如“查看[身份验证指南]”。

### 1.4 确保无障碍

写作时要考虑到所有读者，包括使用屏幕阅读器的残障人士。

### 1.5 面向全球受众

避免使用只有特定文化背景才能理解的成语或隐喻，确保非母语读者也能顺畅阅读。

## 2. 语言与语法

### 2.1 使用第二人称

多用“你（You）”，少用“我们（We）”。这能让读者更有代入感。

### 2.2 坚持主动语态

明确指出动作的执行者。

* ✅ **示例：** “点击按钮”
* ❌ **避免：** “按钮应该被点击”

### 2.3 遵循美式标准

统一使用美式英语的拼写和标点符号习惯。

* ✅ **推荐：** “Color”
* ❌ **避免：** “Colour”

### 2.4 先置条件，后置指令

在给操作指示前先说前提。

* ✅ **推荐：** “要删除文件，请点击红色按钮”
* ❌ **避免：** “点击红色按钮来删除文件”


### 2.5 查阅专用词汇表

遇到特定术语的拼写或用法不确定时，必须对照官方 Word List。

## 3. 格式、标点与组织

### 3.1 标题使用句式大小写（Sentence Case）

除了首字母和专有名词，标题其余部分小写。

### 3.2 规范列表使用

 * **编号列表：** 用于有先后顺序的操作步骤。
 * **点状列表：** 用于无先后顺序的要点罗列。
 * **描述列表：** 用于成对的数据（如：键/值对）。

### 3.3 使用牛津逗号

在并列项的最后一项前也加逗号。例如：A, B, and C。

### 3.4 代码字体应用

所有涉及代码、API、文件名或命令行文本，必须使用 ```code font```。

### 3.5 界面元素加粗

文档中提到的 UI 按钮、菜单项等，应使用**加粗**标注。

### 3.6 消除日期歧义

不要使用```01/02/2026```这种在不同国家理解不同的格式，应写成```Jan 2, 2026```。

## 4. 图片规范

### 4.1 提供替代文本

为每张图片编写描述，方便屏幕阅读器识别。

### 4.2 优先选择高质量图像

在可行的情况下，提供高分辨率图片或矢量图（如 SVG），确保缩放不失真。
