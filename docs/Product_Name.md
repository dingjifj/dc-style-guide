# 产品与功能命名规范

>**页面摘要**：本页详细规定了如何在文档中正确书写、缩写以及引用其它公司的产品、服务与功能名称，以确保品牌一致性。

## 1. 大写规范

* **使用标题大写:** 产品名通常首字母大写（冠词、介词除外）。
  * 例外：若为了匹配 UI 上的特定标签，需遵循 UI 实际显示的大小写。
  
* **尊重官方命名：** 书写任何第三方品牌、软件或开源社区定义的产品/功能时，必须严格遵循其官方大小写规范。
  * 例如：Kubernetes 中的 Pod 和 Job

* **句首特殊处理：** 如果某个产品名官方规定首位字母小写（如 macOS），即便在句首也保持小写。
  * **推荐做法：** 调整句式，避免将此类单词置于句首。  
  * ✅**推荐**：You can use macOS to run...
  * ❌**避免**：macOS can run...

## 2. 功能名称

* **默认小写：** 除非该功能有官方指定的正式名称，否则“功能”名称通常使用小写。
* **不确定时参考先例：** 如果不确定某个功能名是否需要大写，请参考已有的官方文档或对应的 UI 标签。

## 3. 缩写与简化原则

* **坚持使用全称：** 严禁随意缩写公司的注册商标产品名。
  * 例外：为了匹配空间有限的 UI 标签时可以缩写，但需确保语境清晰。
* **使用通用术语替代：** 为了避免不断重复冗长的产品名，可以在首次提及全称（如 Anthos Service Mesh）后，在后续描述中使用通用术语（如 the service mesh）。

## 4. 冠词的使用

| 规则 | 正确示例 | 错误示例 |
|------|----------|----------|
| **产品名前不加 the** | Using Cloud Datastore | Using **the** Cloud Datastore |
| **作为修饰语时加 the** | **The** Cloud Datastore options page |— |
| **工具/API 名前加 the** | **The** gcloud CLI / The Transcoder APIt | — |
| **不定冠词需匹配读音** | **An** Anthos Service Mesh environment | **A** Anthos... |

## 5. 语法禁用

* **禁止名词动词化：** 严禁将产品或功能名称直接当作动词使用。
  * ❌**避免**：Please Google this.
* **“服务”一词的用法：** 可以将产品称为“服务”（如 the Compute Engine service），但如果这会导致语境模糊，请直接使用产品原名。
